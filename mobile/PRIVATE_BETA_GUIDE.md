# Artist Ritual – Private Beta Guide

Artist Ritual은 **아티스트의 작업 순간을 기록하고 작업 습관을 만드는 앱**입니다.

**Artist Ritual은 완성된 작품이 아니라 작업 과정 자체를 기록하는 앱입니다.**

대부분의 작가는 SNS = 완성된 결과 업로드라고 생각합니다. 하지만 이 앱은 다릅니다.

| | |
|---|---|
| Instagram | 결과 |
| Artist Ritual | 과정 |

이번 프라이빗 베타의 목표는 **작가들이 실제로 작업 순간을 기록하는지**, 그리고 **그 기록이 반복되는지**를 확인하는 것입니다.

테스트 기간 동안 자유롭게 사용해 주세요.

---

## 설치

- **iPhone (Expo Go)**: QR 코드 스캔으로 연결
- **iPhone (TestFlight)**: 초대 링크로 설치
- **Android**: APK 다운로드 링크로 설치

자세한 설치 방법은 [BETA_INSTALL_GUIDE.md](./BETA_INSTALL_GUIDE.md)를 참고하세요.

---

## 1. 로그인

첫 화면에서 **Enter**를 누릅니다.

아무 이메일과 아무 비밀번호를 입력해도 됩니다.  
(베타 테스트용 로그인입니다.)

만약 백엔드 API가 연결되어 있지 않으면 자동으로 **Mock Login(모의 로그인)**으로 진행됩니다.

---

## 2. Record Moment (작업 순간 기록)

Artist Ritual의 핵심 기능입니다.

**작업 중인 순간을 기록해 주세요.**

### 기록 방법

1. 사진 촬영 또는 업로드
2. Note (메모)
3. State (작업 상태)
4. Medium (매체)
5. Save

**가능하면 실제 작업 순간을 기록해 주세요.**

### 예시

- 스케치 중인 순간
- 작업실 책상
- 캔버스 진행 과정
- 실험 중인 재료
- 오늘 작업 결과

**완성된 작품이 아니어도 됩니다.**

---

## 3. Feed (피드)

다른 작가들의 작업 순간을 볼 수 있습니다.

현재 베타에서는 일부 기능이 제한될 수 있습니다.

---

## 4. Profile (프로필)

여기에서 확인할 수 있습니다.

- 내가 기록한 작업 순간
- 작업 기록 히스토리
- 알림 설정

---

## 5. 베타 테스트 중 해주면 좋은 것

가능하다면 아래를 시도해 주세요.

- 하루에 한 번 작업 순간 기록
- 최소 3일 이상 기록
- 기록하면서 불편한 점 메모

### 특히 궁금한 것은 다음입니다

- 기록 과정이 번거로운지
- 어떤 순간에 기록하고 싶어지는지
- 기록을 계속하게 되는지

---

## 6. 기술 참고

### 백엔드 API가 없으면

| 기능 | 동작 |
|------|------|
| 로그인 | Mock Login |
| Feed | 제한 |
| 저장 | 일부 기능 제한 |

전체 기능을 사용하려면 백엔드가 실행 중이어야 합니다.

예: `https://art-fair-platform.vercel.app`

로컬에서 백엔드를 실행 중이라면 `mobile` 폴더에 `.env` 파일을 만들고 아래 값을 설정하세요.

```
EXPO_PUBLIC_API_URL=로컬_API_URL
```

## [팀용] CTO 관찰 포인트

### UX 행동 패턴

테스터가 실제로 이렇게 행동하는지 보세요.

| 기대 | 우려 |
|------|------|
| 작업 시작 → 기록 | 완성 작품만 기록 |
| 작업 끝 → 기록 | |

대부분이 **완성 작품만** 기록한다면, 앱의 UX는 아직 **"process recording"이 아니라 "mini portfolio"**로 인식되고 있는 것입니다. 제품 방향을 바꿀 수 있는 중요한 신호입니다.

### 가장 위험한 UX

테스터가 이렇게 느끼면 바로 기록이 줄어듭니다.

> "기록하려면 입력할 게 너무 많네."

**사진 → Note → State → Medium → Save** 구조에서 입력 필드가 많을수록 진입 장벽이 올라갑니다.

**베타에서는 이런 UX가 더 좋습니다.**

- 사진 → Save (최소)
- 나머지는 선택

기록 습관이 먼저고, 데이터 구조는 나중입니다.
