# ROB 베타 테스트 모집 (Ritual + Bridge)

## C 방향에 맞춘 포스팅 (Ritual과 Bridge 모두 언급)

### 버전 1 (가장 균형 좋음) — 추천
```
작가님들,

두 가지를 동시에 해결하고 싶은 앱을 만들고 있습니다.

하루하루 연습하고 기록하는 **Ritual**
그리고 내 전시, 타임라인, 네트워크를 제대로 관리하는 **Bridge**

이제 이 두 가지를 하나의 앱에서 경험할 수 있게 만들고 있어요.

웹에서는 작가 프로필 V2(타임라인·네트워크 중심)를 새로 만들었고,
Expo Go 앱에서는 Ritual(스트릭·모먼트)과 연결된 베타 테스트를 진행 중입니다.

진지하게 활동하시는 작가님들의 솔직한 피드백이 정말 필요합니다.

앱 설치 후 Ritual도 써보시고, 웹 프로필 V2도 확인해 보시고,
불편한 점·좋은 점 모두 말씀해 주세요.

베타 참여하기 → [링크]

#작가 #예술가앱 #스트릭 #작가프로필 #ROB
```

### 버전 2 (감성적)

```
작가님,

웹 프로필 V2는 상당히 완성됐고,

두 가지를 모두 써보시고,
"이렇게 되면 좋겠다"는 의견을 강하게 주실 작가님 찾습니다.

베타 신청 → [링크]

#작가 #작가앱 #기록 #타임라인 #네트워크
```

```
#작가 #아티스트앱 #스트릭 #프로필
```

## 베타 모집 시점 선택지

### 현재 앱 상태 (솔직한 평가)
| 기능 | 앱 내 상태 | 베타 시 필요 수준 |
|------|-----------|------------------|
| 스트릭 / 연습 기록 (Ritual) | ✅ 있음 | OK |
| Add Exhibition (전시 등록) | ❌ 웹에만 | 앱에 간단 버전 필요 ★★★★★ |
| Timeline 보기 | ❌ 웹에만 | 앱에 간단 타임라인 필요 ★★★★★ |
| Network (협업자) | ❌ 웹에만 | 최소 카드 리스트라도 필요 ★★★★ |
| 작가 프로필 V2 | ❌ 웹에만 | WebView 또는 간단 네이티브 ★★★★★ |
| 웹 프로필로 이동 버튼 | ✅ 있음 | OK |

**결론**: Ritual은 어느 정도 있지만, Bridge 핵심 가치를 앱 안에서 거의 느낄 수 없음.

### Option A (추천) — 베타 2~3주 미루기
- Timeline + Add Exhibition을 React Native로 간단히 앱에 구현
- "전시 등록 → 타임라인에 뜨는 것"까지 체험 가능하게 만든 후 모집
- 작가들이 "아, 이게 되네?" 하고 느끼게 한 뒤 베타 진행

### Option B — 지금 모집, 솔직하게 홍보
- "아직 초기 베타라 Bridge 기능은 웹으로 연결됩니다"라고 명확히 밝히고 모집
- Ritual(스트릭) 위주로 먼저 체험하게 유도

### Option C — WebView 적극 도입
- 작가 프로필 V2를 WebView로 앱 안에 띄우기
- 개발 비용 적고 빠르지만, 네이티브 느낌은 떨어짐

## 베타 빌드 시 설정 (선택)

EAS로 preview 빌드할 때, 로그인을 "아무 이메일+비밀번호"로 허용하려면:

```bash
# .env 또는 eas.json에서
EXPO_PUBLIC_BETA=1
```

Expo Go로 실행 시에는 `__DEV__` 모드라 이미 아무 이메일/비밀번호로 로그인 가능합니다.
