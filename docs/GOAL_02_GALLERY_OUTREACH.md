# Goal 02 - Gallery Outreach (Instagram Lead + DM Workflow)

이 Goal은 "갤러리 리드 수집 -> 검증 -> 맞춤 DM 초안 -> 수동 DM 발송" 파이프라인을 만든다.
핵심은 자동화 효율과 함께 플랫폼 정책/법적 리스크를 통제하는 것이다.

## Goal Card

- Goal ID: G-02
- Goal 제목: 갤러리 인스타 리드 수집 및 DM 아웃리치 시스템화
- 기간: 2026-04-13 ~ 2026-04-27 (2주)
- Owner (CEO): Growth Owner
- Tech Owner (CTO): Tech Lead
- 상태: planned

## Why (배경)

- 갤러리 파트너 발굴은 수동 탐색 비중이 높아 속도와 재현성이 낮다.
- 리드 품질 기준과 메시지 실험 프레임이 없으면 결과 학습이 누적되지 않는다.

## Success Metrics (KPI 2~3개)

- KPI 1: 유효 리드 생성 수 (주간)
  - 현재값: TBD
  - 목표값: 주 50개 이상 (중복/저품질 제외)
  - 측정 방법: 리드 시트 상태값(`new`, `qualified`) 카운트
- KPI 2: DM 응답률
  - 현재값: TBD
  - 목표값: 8% 이상
  - 측정 방법: 발송 후 7일 내 응답 비율
- KPI 3: 리드 검증 정확도
  - 현재값: TBD
  - 목표값: 샘플 검수 기준 90% 이상 적합
  - 측정 방법: 수동 샘플 20건 품질 점검

## Scope

### In Scope (이번에 한다)

- 리드 수집 규칙 정의 (지역/장르/활동성 기준)
- 중복 제거, 품질 점수화, 우선순위 큐 자동화
- DM 템플릿 A/B 2종 생성 + 수동 발송 프로세스

### Out of Scope (이번에 안 한다)

- 무차별 대량 발송
- 계정 자동 생성/우회 또는 플랫폼 정책 위반 접근
- 승인 없는 완전 자동 DM 발송

## Compliance and Safety Gates

- 데이터 수집은 플랫폼 정책과 법적 기준을 준수해야 한다.
- 개인정보/민감정보를 불필요하게 저장하지 않는다.
- DM 발송은 아래 조건을 모두 충족해야 한다:
  - [ ] 타겟 적합성 점수 기준 충족
  - [ ] 메시지 톤/사실 검증 완료
  - [ ] 수동 발송자 체크

## Execution Plan (Issue 단위)

- Issue A: 리드 스키마/품질 기준 확정
  - 목적: "좋은 리드" 정의 표준화
  - 완료 기준: 필수 필드, 제외 기준, 점수 룰 문서화
  - 담당: CEO/CTO
- Issue B: 리드 수집 + 중복 제거 파이프라인
  - 목적: 반복 가능한 lead intake 구축
  - 완료 기준: 일 단위 실행 시 중복 제거된 `new` 리드 생성
  - 담당: Engineer
- Issue C: DM 카피 자동 생성기 + 승인 큐
  - 목적: 맞춤 메시지 품질을 유지하며 처리량 확대
  - 완료 기준: 리드별 DM 초안 생성, 승인/반려 상태 기록
  - 담당: Engineer
- Issue D: 실험 대시보드(응답률/차단률)
  - 목적: 메시지 전략 학습 루프 운영
  - 완료 기준: 템플릿별 성과 비교 가능
  - 담당: CTO

## Review Cadence

- 주간 점검 요일: 화요일
- 중간 중단 조건: 정책 위반 위험 신호, 차단률 급증, 민원 발생
- Goal 종료 조건: KPI 3개 중 2개 이상 달성 + 다음 실험 주제 정의

- [x] 성과 학습을 위한 지표 정의가 있다.
- [x] 리드 스키마/이메일 정책 문서가 있다 (`docs/G02_LEAD_SCHEMA_AND_EMAIL_POLICY.md`)
