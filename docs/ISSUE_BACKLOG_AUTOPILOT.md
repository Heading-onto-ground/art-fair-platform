# Issue Backlog (Autopilot)

현재 환경에서는 GitHub CLI(`gh`)와 로컬 `GITHUB_TOKEN`이 없어 이슈를 원격에 직접 생성할 수 없습니다.
아래 이슈를 그대로 GitHub Issues에 등록하면 됩니다.

## G-01: Platform Stability

### 1) [G-01/A] CI 실패 분류 규칙 수립

- 배경/목표: CI 실패 원인 표준화로 재발 제어
- 완료 기준:
  - 실패 태그 체계(`ci:web`, `ci:mobile`, `ci:infra`) 문서화
  - 첫 주 1회 적용 기록
- 제외 범위: 테스트 프레임워크 교체

### 2) [G-01/B] PR 검증 체크리스트 표준 적용

- 배경/목표: 머지 전 검증 누락 방지
- 완료 기준:
  - PR 체크리스트 문서 또는 템플릿 반영
  - 최소 검증 항목 합의 (`lint:web`, `typecheck`, `test`)
- 제외 범위: 무관한 리팩터링

### 3) [G-01/C] lint gate 확장 후보 정리

- 배경/목표: CI 품질 게이트 점진 확대
- 완료 기준:
  - 후보 경로 1~2개 선정
  - 선정 경로 lint 정리 완료
- 제외 범위: 전체 lint debt 일괄 청산

### 4) [G-01/D] 주간 CI 리포트 발행

- 배경/목표: KPI 기반 주간 의사결정
- 완료 기준:
  - KPI 3개 포함한 1페이지 리포트 1회
  - 다음 주 액션 1~3개
- 제외 범위: 대시보드 풀 오토메이션

## G-02: Gallery Outreach

### 5) [G-02/A] 리드 스키마/품질 기준 확정

- 배경/목표: 유효 리드 정의 표준화
- 완료 기준:
  - 필수 필드/제외 기준/점수 룰 문서화
- 제외 범위: 무차별 대량 수집

### 6) [G-02/B] 리드 수집 + 중복 제거 파이프라인

- 배경/목표: 반복 가능한 lead intake 구축
- 완료 기준:
  - 일 단위 실행 시 중복 제거된 `new` 리드 생성
- 제외 범위: 정책 위반 가능 자동화

### 7) [G-02/C] DM 초안 생성 + 승인 큐

- 배경/목표: 메시지 품질 유지하며 처리량 확대
- 완료 기준:
  - 리드별 DM 초안 생성
  - 승인/반려 상태 기록
- 제외 범위: 승인 없는 자동 발송

### 8) [G-02/D] 응답률 실험 리포트

- 배경/목표: 템플릿 성과 학습 루프 구축
- 완료 기준:
  - 템플릿별 응답률 비교 가능
- 제외 범위: 전 채널 통합 대시보드

## G-03: Artist SNS Marketing

### 9) [G-03/A] 채널별 ICP/메시지 맵 정리

- 배경/목표: 타겟별 메시지 일관성
- 완료 기준:
  - 채널 x 페르소나 x CTA 매트릭스 문서화
- 제외 범위: 브랜드 전면 리뉴얼

### 10) [G-03/B] 콘텐츠 캘린더 생성 루프

- 배경/목표: 주간 운영 사이클 자동화
- 완료 기준:
  - 1주 분량 캘린더 자동 생성 + 리뷰 절차
- 제외 범위: 영상 제작 자동화 전체 대체

### 11) [G-03/C] 모집 랜딩/폼 A-B 실험

- 배경/목표: 전환율 개선
- 완료 기준:
  - 카피/CTA 2개 이상 실험 설계 및 기록
- 제외 범위: 대규모 유료광고 자동 집행

### 12) [G-03/D] 주간 마케팅 리포트 자동화

- 배경/목표: 반복 학습 속도 향상
- 완료 기준:
  - 유입/전환/콘텐츠 성과 1페이지 리포트
- 제외 범위: BI 도구 대규모 도입
