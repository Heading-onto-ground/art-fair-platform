# AI Agent Operating Rules

이 문서는 "에이전트를 회사처럼 운영"하기 위한 최소 운영 규칙입니다.
프로젝트 현실에 맞춰 경량으로 시작하고, 데이터가 쌓이면 확장합니다.

## 1) 운영 원칙

- 목표 우선: 모든 작업은 활성 Goal과 연결되어야 한다.
- 작은 배치: 작업 단위는 1~2일 내 검증 가능한 크기로 쪼갠다.
- 근거 기반: 결정은 추정이 아니라 로그, 테스트, 사용자 반응으로 평가한다.
- 안전 우선: 인증/권한/결제/개인정보 관련 작업은 리뷰 강도를 높인다.
- 비용 통제: 실패 루프를 제한하고 사람 승인 지점을 명확히 둔다.

## 2) 역할 정의 (경량 3역할)

### CEO (Product Owner)

- 분기/월 Goal 설정 및 우선순위 결정
- "무엇을 왜 하는지"를 3문장으로 명확화
- 승인 권한: 새 Goal 생성, 기존 Goal 중단/변경

### CTO (Tech Lead)

- 실행 전략 결정 (아키텍처, 리스크, 범위)
- 완료 기준(Definition of Done) 설정
- 승인 권한: 기술 방향 결정, 롤백/핫픽스 지시

### Engineer (Executor)

- 구현, 테스트, 문서 반영, 변경 리포트
- 실패 원인과 다음 액션 제시
- 승인 권한: DoD 충족 시 머지 요청

## 3) 의사결정 규칙

- 우선순위 충돌 시: CEO 결정
- 기술 방향 충돌 시: CTO 결정
- 긴급 장애 시: CTO 즉시 대응, CEO 사후 공유
- 30분 이상 막히면: 단독 진행 금지, 이슈로 전환 후 결정 요청

## 4) 작업 정책

- 모든 작업은 Issue 1개와 연결
- Issue에는 반드시 아래 3가지 포함:
  - 배경/목표
  - 완료 기준(테스트 포함)
  - 제외 범위(이번에 하지 않을 것)
- "좋아 보이는 개선"은 별도 Issue로 분리 (범위 확장 금지)

## 5) 비용/실패 통제

- 1회 작업 제한:
  - 탐색/분석: 최대 30분
  - 구현 시도: 최대 2회
  - 테스트 실패 반복: 최대 2회 후 중단
- 중단 기준 도달 시:
  - 현재 상태 요약
  - 실패 원인 가설 1~2개
  - 필요한 승인/결정 1개

## 6) 완료 기준 (Definition of Done)

아래를 모두 만족해야 "완료"로 본다.

- 코드 변경이 목표와 직접 연결된다.
- 관련 테스트/검증을 실행했고 결과를 기록했다.
- 리스크가 있는 영역은 리뷰 포인트를 명시했다.
- 변경 이유를 3줄 내로 설명 가능하다.

## 7) 운영 리듬 (권장)

- 주간 1회: Goal 점검 (진행/중단/추가)
- 일간 1회: 실행 상태 공유 (막힘/결정 필요/다음 액션)
- 릴리즈 전: CI 기준 체크리스트 검증

## 8) 외부 아웃리치 자동화 가드레일

아래는 SNS/DM/이메일 아웃리치 자동화에 대한 최소 안전 규칙이다.

- 이메일 자동 발송 허용: 수신 거부, 빈도 제한, 발송 로그를 반드시 적용한다.
- DM 수동 발송 원칙: DM은 사람이 직접 발송하고, 에이전트는 초안/대상 큐까지만 자동화한다.
- 정책 준수 우선: 플랫폼 약관/법적 제한을 위반할 수 있는 자동화는 중단한다.
- 최소 수집 원칙: 목적과 무관한 개인정보를 저장하지 않는다.
- 차단 신호 대응: 신고/차단/클레임 증가 시 즉시 캠페인을 멈춘다.
- 품질 우선: 대량 발송보다 타겟 적합성과 메시지 정확도를 우선한다.
